# UI 动效与排版统一规范

本规范是全项目动画与字体排版的唯一标准。目标是：同一种交互在任何场景里
看起来、动起来都一样。所有 token 定义在 `src/styles/base.css` 的 `:root`，
场景样式只允许引用 token，不得再写裸数值（例外见下文）。

## 1. 动效语言

本项目是像素风：缓动一律使用 `steps()` 步进，禁止在过渡中出现平滑缓动
（`ease`、`ease-in`、`ease-out`、`ease-in-out`、`cubic-bezier(...)`）。
逐帧精灵动画（`background-position` / 换图的 `steps(N)` keyframes）本身就是
像素语言，保持原样，不属于"平滑缓动"。

### 时长 Token

| Token | 值 | 用途 |
| --- | --- | --- |
| `--dur-press` | `90ms` | 按压、勾选等即时微反馈 |
| `--dur-fast` | `120ms` | 小控件状态切换（hover、开关、标签切换） |
| `--dur-med` | `180ms` | 默认 UI 过渡（卡片、面板、列表项、提示） |
| `--dur-slow` | `240ms` | 弹层、抽屉、遮罩、对话框入场 |
| `--dur-scene` | `320ms` | 场景级转场与大型编排的单拍 |

映射规则（修改旧值时）：

- `≤ 100ms` → `--dur-press`
- `101–140ms` → `--dur-fast`
- `141–200ms` → `--dur-med`
- `201–280ms` → `--dur-slow`
- `281–400ms` → `--dur-scene`
- `> 400ms`：只允许用于叙事编排（红闪、黑屏、打字机、剧情倒计时），
  保留裸值并在提交说明中标注理由；普通 UI 不得使用。

### 缓动 Token

| Token | 值 | 用途 |
| --- | --- | --- |
| `--ease-press` | `steps(2, end)` | 按压/即时反馈 |
| `--ease-pixel` | `steps(3, end)` | 默认过渡 |
| `--ease-in-pixel` | `steps(4, end)` | 入场（配合位移动画） |
| `--ease-out-pixel` | `steps(2, end)` | 出场/消失 |

规则：

- `transition` 中的平滑缓动全部替换为 `var(--ease-pixel)`（入场用
  `--ease-in-pixel`）。
- 进度条、加载条、走马灯等连续运动保留 `linear`。
- 已有的 `steps(N, end)` 过渡：N 不在 2–5 之间时归一到上表最近档位。

### 标准动作模式

- **入场（浮层/卡片/对话框/Toast）**：`opacity 0→1 + translateY(16px→0)`，
  `--dur-slow` + `steps(4, end)`，即全局 `toast-in` 的视觉语言。新入场动画
  一律复用该节奏，不要发明新的位移距离。
- **按压**：`transform: translateY(2px)`（大按钮 `3px`），`:active` 即时生效，
  不加 transition；需要回弹过渡时用 `--dur-press` + `--ease-press`。
- **循环装饰动画**（闪烁、漂浮、呼吸）：保留原时长，只统一缓动写法。
- **帧震屏**：复用 `frame-shake`（`0.12s steps(2, end)`）语言，不新增平移距离
  不同的震屏 keyframes。

## 2. 字体排版 Token

| Token | 值 | 用途 |
| --- | --- | --- |
| `--fs-micro` | `9px` | 角标、状态栏小字、徽标 |
| `--fs-caption` | `11px` | 辅助说明、时间戳、次要信息 |
| `--fs-body` | `13px` | 正文、列表、对话 |
| `--fs-label` | `14px` | 按钮、强调标签、小标题 |
| `--fs-title` | `18px` | 页面/区块标题 |
| `--fs-display` | `24px` | 大数字、展示位 |

规则：

- 语义角色相同的文字（页标题、正文、辅助说明、角标、按钮）在任何场景
  必须使用同一个 token。
- 现有尺寸落在某 token ±1px 内且角色相同 → 直接归入该 token。
- 孤立奇数尺寸（7、8、15、17、19、21、22、23、25、27、29px 等）→ 归入最近的
  token；确因像素对齐必须保留的，在提交说明中写明理由。
- 叙事大标题（> 30px 的展示字、剧情数字）保留裸值，但同一元素的多个
  实例必须一致。
- 禁用 `rem`/视口单位表达场景内文字（`clamp()` 仅允许用于跨桌面/移动
  自适应的剧情大字）；场景内一律 `px` token。
- 行高：正文 `1.5–1.65`，紧凑标签/列表 `1.2–1.45`，标题 `1.05–1.3`；
  同一角色在不同场景行高必须一致。
- 字重：正文 `400/700`，标题与按钮 `700/900`；同一角色保持一致。
- `font-family`：界面与中文一律 `var(--font-pixel)`；`monospace` 只保留给
  数字读数、计时器、代码段等刻意数码风的元素。

## 3. 表现层逻辑一致性

逐场景自查以下各项，对照 `src/scenes/phone/registry.tsx` 的 contract 与
各场景 `README.md`：

- 顶部 `40px` 全局状态栏保留区不被场景内容遮挡或复用。
- 场景的 加载 → 内容 → 结果 阶段顺序与文案和 contract 一致。
- 返回入口（返回手机主页 / 返回上一级）存在且行为一致。
- 锁定位遵守 `AGENTS.md` 的 Locked Feature Visual Contract（静态 `xxx`、
  无按钮语义、无辅助文案）。
- **禁止改动**：控制器、状态机、谜题数值、门禁条件、存档字段。发现需要
  控制器层面的逻辑问题时只记录报告，不改代码。

## 4. 验证

- 每个场景改完运行 `npm run typecheck`。
- 交付前在 `430×860` 逻辑视口目视检查：无横向溢出、无文字截断、动画节奏
  符合上表。
