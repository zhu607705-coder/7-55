# 图书馆期末周关卡设计

## 目标

- 把“离座后座位被预约”“考后求是潮自行车拥堵”“CC98 十大与资料检索”组成一条完整关卡链。
- 复用现有图书馆、校园卡、浙大体艺、CC98 四套页面，不新增整页背景图。
- 游戏逻辑只维护状态与领域事件。旁白文本、音频文件、延迟、音量和配乐变化由独立配置维护。
- 复用范围限定为场景画面与页面结构。不同剧情阶段使用差异明显的旁白演绎、配乐和音效。

## 关卡流程

1. 玩家在图书馆页面预约座位，点击“离开 3 分钟”。返回后原座位显示为已被预约。
2. 页面提示需要临时离座凭证。玩家进入校园卡，完成身份校验。
3. 校园卡把通行提示指向浙大体艺。体艺页面显示考后求是潮自行车拥堵，点击后记录绕行信息。
4. CC98 出现一条排名 01 的帖子：“考完试求是潮全是自行车，这不值得十大吗”。
5. 玩家打开 CC98 搜索。即使关键词有错字，仍返回图书馆临时离座规则、绕行路线和复习资料。
6. 玩家采用“图书馆临时离座与违约处理汇总”，回到选座页刷新，恢复预约并完成关卡。

## 荒诞与嘲讽

- 旁白只评价当前结果，不向玩家解释操作步骤。
- 嘲讽对象是规则、手续和信息分散，避免直接贬低玩家。
- 关键句包含“这不值得十大吗”，并让帖子实际进入排名 01。
- CC98 搜索在输入不完整或有错字时仍返回资料，形成明确的功能反差。

## 素材复用矩阵

| 素材 | 原用途 | 本关用途 | 复用方式 |
| --- | --- | --- | --- |
| 图书馆选座图 | 选座演示 | 预约、丢座、恢复预约 | 同一底图切换状态标签与按钮 |
| 校园卡页 | 数字 0 收集 | 临时离座身份校验 | 增加一条任务提示，不改主卡面 |
| 浙大体艺页 | 数字 7 收集 | 求是潮自行车通行提示 | 复用主图，叠加交通状态条 |
| CC98 热门页 | 论坛展示 | 十大帖子与资料检索 | 复用列表、详情和编辑系统 |
| 现有 UI 音效 | 点击、错误、通知 | 预约失败、校验、帖子提示 | 在新事件上重新组合 |
| MiniMax 分阶段配乐 | 新增 | 预约、丢座、堵车、十大、检索、恢复 | 每个阶段使用独立配乐，画面不变时依靠声音区分状态 |
| MiniMax 分阶段音效 | 新增 | 丢座、车铃、十大、检索 | 从 MiniMax 音效设计结果切分为独立 cue |

页面级素材复用率为 4/4，新增整页视觉素材为 0。音频不计入复用率。旁白保持同一角色声线，通过语速、音高和停顿区分阶段。

## 阶段音频差异

| 阶段 | 配乐方向 | 旁白演绎 |
| --- | --- | --- |
| 预约成功 | 低速、稀疏、钟表节拍 | 语速较慢，结尾保留停顿 |
| 座位丢失 | 干燥故障节奏、短促错误音 | 语速提高，音高略升 |
| 自行车堵车 | 高频车铃、快速打击节奏 | 语速最快，句间停顿缩短 |
| CC98 十大 | 强提示音、排名上升节奏 | 重读“十大”和“条件齐了” |
| CC98 检索 | 键盘点击、扫描脉冲 | 语速中等，重读检索结果 |
| 座位恢复 | 低速明亮和弦、短确认音 | 语速降低，音高略降 |

## 状态与事件

状态顺序：

`idle -> seat_reserved -> seat_lost -> pass_validated -> bike_jam -> top_ten -> guide_found -> seat_recovered`

领域事件：

- `library_finals_started`
- `library_seat_lost`
- `library_pass_validated`
- `library_bike_jam_reported`
- `cc98_top_ten_opened`
- `cc98_material_found`
- `library_seat_recovered`

音频层只订阅这些事件，不回写关卡状态。音频播放失败时，关卡仍可继续。

## 音频对轴规则

- 每个事件的 cue 使用独立 `offsetMs`。
- 旁白时长由 `ffprobe` 写入生成清单，字幕按实际文件时长显示。
- 配乐压低由旁白开始和结束控制，不参与关卡完成判断。
- 剧情改写时修改 `src/data/library-finals.content.json`，再运行 `npm run audio:library-finals -- --force`。
- 关卡机制不读取旁白文本，不等待旁白结束，也不依赖音频文件长度。
