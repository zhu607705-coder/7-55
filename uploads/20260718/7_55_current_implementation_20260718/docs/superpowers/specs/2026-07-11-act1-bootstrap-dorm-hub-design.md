# 第一章启动链与宿舍据点设计

日期：2026-07-11  
确认方案：A，宿舍为可反复返回据点

## 目标

玩家首次进入 RPG 时只能看到人物、校园地图和临时道具栏。人物暂时无法移动。玩家必须在既有手机应用中依次完成四项设置，RPG 操作才会完整开放：

1. 在浙大登录页完成身份登录，账号信息从浙大钉读取。
2. 在浙大钉部门黄页拨打联络电话，电话号码使用剧情学号。
3. 在 CC98 交换区取得方向键和交互键。
4. 在浙大体艺开始一次锻炼记录，解锁人物行走。

四项设置完成后，玩家返回 RPG，开始第一关跑图。

## 屏幕方向

- 手机应用始终使用 `430×930` 竖屏逻辑尺寸，禁止旋转。
- RPG 使用固定横屏画布；切换至 RPG 时由运行时更换布局。
- 手机和 RPG 共享同一进度状态，切换方向不重载页面。
- 横屏 RPG 需要保留安全区，触控操作键不能覆盖人物、任务目标和场景标签。

## 第一关流程

### 1. 无操作状态

首次进入只显示人物、地图、临时道具栏和“打开手机”入口。方向键与点击移动均无效。任务栏只显示当前缺少的第一项设置。

### 2. 登录

浙大登录页提供账号栏和读取浙大钉信息的按钮。玩家无需输入真实账号；剧情学号使用虚构配置值。成功后产生 `act1_identity_verified`。

### 3. 联络联网

浙大钉部门黄页显示多个虚构部门号码。玩家从个人信息中读取剧情学号，并拨打匹配号码。成功后产生 `act1_phone_linked`，RPG 显示网络状态。

### 4. 安装操作键

CC98 交换帖列出若干虚构交换物。玩家用登录回执交换“方向键”和“交互键”。成功后产生 `act1_controls_installed`。此时按键出现，但人物仍无法移动。

### 5. 开始锻炼

浙大体艺页面要求点击“开始锻炼”。成功后产生 `act1_movement_enabled`。返回 RPG 后方向键、键盘和触控移动全部生效。

### 6. 强制道具

第一关地图固定放置一个主线道具。旁白在玩家经过目标附近仍未拾取时给出直接提示。取得后产生 `act1_required_item_collected`。

建议道具：`游戏卡带`。它承担第一章入口凭证，并在后续章节的“游戏”菜单内保留为剧情记录。

### 7. 道具栏收纳

玩家取得主线道具并通过第一关后，屏幕侧边的临时道具栏隐藏。道具、线索和任务记录统一迁入暂停菜单中的“游戏”页。后续章节不再显示常驻侧边道具栏。

### 8. 全图跑图

第一关地图设置若干必要区域。玩家必须进入每个区域一次；已访问区域在地图中显示短标签。完成跑图且持有主线道具后，“游戏”入口开放。

### 9. 进入第二章

玩家从“游戏”入口进入第二章。第二章主要循环为：

- 在手机应用中收集情报、凭证和道具。
- 在横屏 RPG 中操作人物完成路线与场景任务。
- 在图书馆 8 段跨应用剧情中处理选座、CC98、体艺、照片证据和恢复座位。

## 宿舍据点

- 第一章结尾首次开放，后续章节可反复返回。
- 固定横屏俯视场景，房间居中，场外黑色。
- 可整理“游戏”页中的道具和线索。
- 可查看室友留言、章节目标和已完成支线。
- 同一场景通过灯光、桌面物件、留言和可用交互更新阶段，不重复制作整页背景。
- 宿舍出口连接校园地图；手机入口随时可用。

## 状态模型

新增 `ActOneBootstrapState`：

```ts
interface ActOneBootstrapState {
  phase:
    | "identity_required"
    | "phone_link_required"
    | "controls_required"
    | "movement_required"
    | "item_required"
    | "map_required"
    | "chapter_two_ready"
    | "complete";
  identityVerified: boolean;
  phoneLinked: boolean;
  controlsInstalled: boolean;
  movementEnabled: boolean;
  requiredItemCollected: boolean;
  visitedAreaIds: string[];
  gameMenuUnlocked: boolean;
  dormHubUnlocked: boolean;
}
```

推进只允许按顺序发生。重复操作返回成功状态或给出短反馈，不回退进度。

## 文本与口吻

- 所有任务文案、旁白字幕、交换帖文本、部门名称和剧情学号存放在 JSON。
- React 与 Phaser 组件只读取文本 ID，不内嵌长剧情句。
- 第一章旁白保持现有角色，语气直接、短句、略带嘲讽。
- 第二章图书馆旁白继续使用英文，改用独立女声。
- 笑点来自操作步骤、等待时间、论坛行为和系统反馈，避免长篇制度解释。

第一章示例：

- “先登录。这个角色目前连移动权限都没有。”
- “方向键装好了。体艺还没同意你走路。”
- “地图上已经画了一个大标记。请把那个道具捡起来。”

第二章英文示例：

- “You were gone for three minutes. The reservation system needed less.”
- “Eighteen ac01 replies removed. One usable sentence remains. Historic.”
- “Three replies later, the post is number one. The traffic is unchanged.”

## 音频

- 第一章和第二章使用独立旁白声音配置。
- 第二章 `library-finals` 事件映射到英文女声文件。
- 每个阶段的字幕、音频文件、开始偏移和 ducking 写入音频 JSON。
- 音频只订阅领域事件，播放失败不影响状态推进。
- 同一场景的不同阶段需要明显不同的配乐、节拍和反馈音。

## 验收

- 初次进入 RPG 无法移动，且能打开手机完成四项解锁。
- 四项解锁严格按顺序推进，返回 RPG 后移动立即生效。
- 第一关必须取得主线道具并完成所有必要区域访问。
- 第一关结束后常驻道具栏隐藏，“游戏”页可查看已有内容。
- “游戏”入口只在道具与跑图条件都满足时开放。
- 宿舍可以反复进入、离开，并保留阶段状态。
- 手机场景固定竖屏，RPG 场景固定横屏。
- 第二章图书馆字幕为英文，音频为独立女声。
