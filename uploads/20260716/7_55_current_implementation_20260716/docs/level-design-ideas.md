# 《7:55》浙大特色荒诞关卡候选池 v0.1

## 设计方向

### 方案 A：手机界面元交互

按钮、状态栏、输入框、通知、图标和文字都可以成为可拖动或可组合的游戏对象。优点是与现有手机框架衔接直接，玩家容易理解操作边界；风险是连续使用会形成重复感。

### 方案 B：校园流程规则错位

选课、签到、校园网、图书馆预约、校园卡和失物招领维持真实流程外观，但规则之间产生可验证的矛盾。优点是浙大辨识度高；风险是过度依赖校内经验时，外部玩家可能看不懂。

### 方案 C：跨应用道具链

一个应用中获得的界面元素，在另一个应用中解决问题。例如从天气页取走水滴，在相册里清理模糊验证码。优点是能形成章节连续性；风险是道具线过长会增加回溯成本。

推荐采用 `A + B` 作为单关核心，使用少量 `C` 串联章节。每关只保留一个主要规则变化，并在失败后提供可见反馈。

## 单关节奏模板

1. 先展示一个看似正常的校园应用任务。
2. 玩家执行常规操作后获得明确失败反馈。
3. 页面暴露一个可移动、可编辑或可组合的界面元素。
4. 玩家利用该元素改变另一条系统规则。
5. 过关结果保留一个轻量道具或数字，供后续关卡复用。
