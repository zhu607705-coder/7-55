# 《7:55》当前实现审阅说明

## 直接查看

1. 解压 ZIP。
2. 直接打开 `demo/index.html`。
3. 首屏应显示“早八闹钟”、`07:55` 和“开始游戏”。

`demo/index.html` 是单文件离线版本，脚本、样式、字体、图片和音频均已内嵌。离线评审入口默认显示折叠的 `DEV` 检查点，遇到旧存档或异常页面可直接恢复；展示模式可使用 `?dev=0` 隐藏。

## 源码位置

- `src/`：游戏状态、章节流程、React 手机界面、Phaser RPG、任务栏、道具和音频逻辑。
- `src/**/*.test.*`：功能权限、第一章回归、双屏、上传、道具生命周期、语音职责和组件交互测试。
- `scripts/`：地图、美术和音频生成或校验脚本。
- `docs/`：章节剧情及开发说明。
- `public/`、`pageexample/`：运行资源与界面参考素材。
- `progress.md`：按时间记录的实现、测试、浏览器验收、文件大小和哈希。

## 本轮重点

- 章节清晰分隔，功能按剧情事实逐步开放。
- 手机、RPG 和桌面双屏共用当前任务、进度与三档提示。
- 手机任务键可用鼠标、触摸、Enter 和 Space 打开；抽屉明确显示“当前任务”“当前进度”和下一步目标。
- 手机统一采用 `430×860` 逻辑画布；第一章已取得的签到数字按原位置同步显示在任务键、任务抽屉和物品栏，未知位保持 `?`。
- CC98、浙大钉和图书馆的锁定入口保留原卡片框与图标，隐藏名称且不可聚焦、不可点击；`xxx` 只用于文字型未开放内容。
- 浙大钉无用途的底部蓝色占位横条已删除。
- PC 宽屏同时显示手机和地图，焦点决定输入归属。
- 纸质道具可展开，CC98 与恢复申请支持拖拽上传和点击选择。
- 道具在完成最后用途后消费，旧存档会清理失效道具。
- 男旁白与女系统声线分离；操作吐槽和任务提示保持纯文字。
- 手机主页底部天气通知在解锁后可直接进入天气页。
- 校园地图体育场地块已改为正俯视像素贴图。
